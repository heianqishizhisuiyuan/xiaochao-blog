# 博客系统（首版）QA / 验收标准清单

> 适用：首版上线前功能验收 + 回归自测
>
> 技术栈参考：Vue 3 + TypeScript + Vite（当前仓库结构）
>
> 说明：本清单按【P0 必须通过 / P1 建议通过 / P2 优化项】标记优先级；若某功能在当前版本不存在，可标记为 N/A。

---

## 0. 测试准备（环境 / 数据 / 基线）

- [ ] 【P0】明确测试环境
  - [ ] 访问地址（dev / staging / prod）可用
  - [ ] 构建方式明确：`npm run build` + `npm run preview`（或线上部署）
  - [ ] 明确 API / 内容来源（本地 JSON / CMS / 静态 Markdown 等）
- [ ] 【P0】准备测试数据集（至少覆盖）
  - [ ] 文章数：0 篇（空态）、1 篇、>20 篇（分页/列表压力）
  - [ ] 文章内容：纯文本、长文（>5000 字）、多标题层级（H2/H3/H4）、包含代码块、引用、列表、表格（如支持）、图片（横图/竖图/大图）、外链、站内链接
  - [ ] 特殊字符：emoji、中文、英文、混排、符号（`< > & "`）、多空格、换行
- [ ] 【P0】浏览器与设备覆盖（最低）
  - [ ] 桌面端：Chrome（最新版）/ Safari（如有 Mac）/ Edge（最新版）
  - [ ] 移动端：iOS Safari / Android Chrome
  - [ ] 分辨率：375×667、768×1024、1280×800、1440×900（至少 3 档）
- [ ] 【P0】验收基线：无阻断问题
  - [ ] 页面可用、核心路径无崩溃
  - [ ] 控制台无红色报错（可允许第三方脚本 warning，但需记录）
  - [ ] 关键交互有加载态/错误态/空态

## 1. 首页（Home / 文章列表页）

### 1.1 基础展示
- [ ] 【P0】首屏可打开，布局不破版

- [ ] 【P0】文章列表渲染正确
  - [ ] 标题、摘要（如有）、发布时间、作者（如有）、标签/分类（如有）展示与数据一致
  - [ ] 列表项可点击进入文章详情
- [ ] 【P0】空态正确
  - [ ] 无文章时显示友好提示，不出现报错/空白页
- [ ] 【P1】加载态正确
  - [ ] 首次加载/翻页时有 Skeleton/Spinner（或至少按钮禁用/提示）

### 1.2 列表交互（按实际功能勾选）
- [ ] 【P0】分页可用（若有）
  - [ ] 下一页/上一页/页码跳转正确
  - [ ] 切页后滚动位置符合预期（一般回到列表顶部）
  - [ ] URL（query）可反映当前页（如产品定义需要）
- [ ] 【P0】搜索可用（若有）
  - [ ] 关键字搜索结果正确
  - [ ] 无结果时空态正确
  - [ ] 清空搜索后回到原列表
- [ ] 【P1】筛选/分类/标签可用（若有）
  - [ ] 选择筛选条件后列表刷新正确
  - [ ] 多条件组合（如支持）行为正确
  - [ ] URL 可分享复现筛选条件（如产品定义需要）
- [ ] 【P2】排序（最新/最热等）正确（若有）

### 1.3 导航与页脚
- [ ] 【P0】全局导航可用（首页/归档/关于等，按实际）
- [ ] 【P0】外链打开策略正确
  - [ ] 外链 `target=_blank` 时具备安全属性（建议：`rel="noopener noreferrer"`）
- [ ] 【P1】页脚信息（版权、备案、社交链接等）正确

## 2. 文章详情页（Post / Article）

### 2.1 页面结构与渲染
- [ ] 【P0】通过首页点击可进入文章页
- [ ] 【P0】URL 可直接访问（刷新不 404；前端路由部署需验证）
- [ ] 【P0】标题、发布时间、作者、分类/标签显示正确（按实际字段）
- [ ] 【P0】正文渲染正确
  - [ ] 段落、列表、引用、加粗/斜体、链接显示正常
  - [ ] 图片加载正常；失败时有兜底（broken 图片不撑爆布局）
  - [ ] 代码块（若有）换行/滚动正常，不溢出
  - [ ] 超长英文/长 URL 不撑破布局
- [ ] 【P0】文章不存在（404）处理正确
  - [ ] 显示「文章不存在」等提示
  - [ ] 提供返回首页/继续浏览入口

### 2.2 阅读体验（按实际功能勾选）
- [ ] 【P1】目录（TOC）正确（若有）

- [ ] 【P2】阅读进度条/预计阅读时间（若有）正确

### 2.3 交互与状态
- [ ] 【P0】页面加载过程无闪烁/大范围布局抖动（CLS 可感知）
- [ ] 【P0】返回列表后状态符合预期（滚动位置/筛选条件/分页保留，按产品定义）
- [ ] 【P1】分享按钮（若有）

## 5. 性能（首版可落地指标）

> 目标：用户可感知的加载速度与交互流畅。

### 5.1 构建与资源体积
- [ ] 【P0】`npm run build` 可成功（无类型错误、无构建失败）
- [ ] 【P0】首屏资源加载无明显卡顿
- [ ] 【P1】首屏 JS/CSS 体积不过大（建议用浏览器 Network 观察：主 bundle 不应异常膨胀）
- [ ] 【P2】按路由/组件懒加载（如有多页面/多路由）

### 5.2 Web Vitals（建议用 Lighthouse / PageSpeed）
- [ ] 【P1】LCP（最大内容绘制）表现可接受（移动端不明显“等半天”）
- [ ] 【P1】CLS（布局偏移）接近 0（无明显跳动）
- [ ] 【P1】INP/TBT（交互延迟/阻塞）点击、滚动无明显掉帧

### 5.3 图片与静态资源
- [ ] 【P1】图片尺寸与展示匹配（避免 4000px 大图缩成 300px）
- [ ] 【P1】图片懒加载（长文章多图时）
- [ ] 【P2】使用现代格式（WebP/AVIF）与缓存策略（上线配置项）

## 6. SEO（静态站/SPA 场景）

> 若为纯 SPA，SEO 天花板有限：至少保证基础 meta、可分享卡片、站点地图；如需强 SEO，需 SSR/预渲染方案（此处作为验收项记录，不强制要求实现）。

### 6.1 基础 SEO
- [ ] 【P0】每个页面有合理的 `<title>`
  - [ ] 首页 title：包含站点名/主题
  - [ ] 文章页 title：包含文章标题 + 站点名
- [ ] 【P1】meta description（若有）
  - [ ] 首页描述清晰
  - [ ] 文章页描述为摘要/前 N 字
- [ ] 【P1】规范化链接 canonical（若有）避免重复收录

### 6.2 可抓取性
- [ ] 【P1】`robots.txt` 存在且策略正确（至少不误封全站）
- [ ] 【P1】`sitemap.xml` 存在且可访问（文章 URL 完整）
- [ ] 【P1】404/重定向策略合理（不存在页面返回 404，不返回 200）

### 6.3 社交分享卡片
- [ ] 【P1】Open Graph（OG）标签（`og:title/og:description/og:image/og:url`）正确（若配置）
- [ ] 【P2】Twitter Card 标签正确（若配置）

### 6.4 结构化数据（可选）
- [ ] 【P2】文章页包含 `BlogPosting` / `Article` JSON-LD（若追求 SEO）

## 7. 无障碍（Accessibility / A11y）

- [ ] 【P0】键盘可用
  - [ ] Tab 可遍历主要可交互元素
  - [ ] 聚焦样式明显（不要把 outline 全关）
  - [ ] Enter/Space 可触发按钮
- [ ] 【P0】语义结构合理
  - [ ] 页面有且仅有一个 H1（通常为文章标题或首页主标题）
  - [ ] 标题层级不跳级（H2 → H3…）
- [ ] 【P1】图片 alt
  - [ ] 内容图片有意义的 `alt`（装饰性图片可空 alt）
- [ ] 【P1】对比度
  - [ ] 正文文字与背景对比度足够（至少接近 WCAG AA）
- [ ] 【P1】链接可识别
  - [ ] 仅颜色区分的链接需有下划线或 hover/focus 提示
- [ ] 【P2】跳到正文（Skip to content）

## 8. 安全与合规（首版最低线）

> 本项不做“渗透测试”，仅做上线最低风险检查。

- [ ] 【P0】外链打开新窗口时具备 `rel="noopener noreferrer"`（防 tabnabbing）
- [ ] 【P0】无明显 XSS 注入点（富文本/Markdown 渲染需特别关注）
  - [ ] 文章内容中的 `<script>` / `onerror=` 等不会被执行
- [ ] 【P1】依赖与构建产物不暴露敏感信息
  - [ ] 不把 `.env`、密钥、私有 API BaseURL 等打进前端（可通过搜关键字自查）

## 附：快速自测流程（最小闭环）

1. 本地：`npm run build`（确保可发版）
2. 本地：`npm run preview` 打开首页 → 进入文章页 → 刷新 → 404 页
3. 移动端：至少验证 375 宽首页与文章页
4. 浏览器控制台：确保无红色报错
5. Lighthouse（可选但推荐）：首页与文章页各跑一次，记录主要指标
