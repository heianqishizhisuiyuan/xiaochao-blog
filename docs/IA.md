# 博客系统信息架构（IA）与页面清单

> 目标：为当前 `blog/`（Vue3 + TS + Vite + Element Plus + Pinia + Vue Router）提供**产品定位 + 信息架构 + 页面树 + 关键页面区块**的设计文档，作为后续路由、组件拆分、接口/数据模型与迭代规划的依据。

---

## 1. 站点定位（Positioning）

### 1.1 站点目标
- **对外展示**：承载作者的长期内容资产（技术/产品/随笔），形成可持续的个人品牌与可信度。
- **对内生产**：提供轻量但完整的内容生产与管理能力（写作、草稿、发布、修订、归档）。
- **可增长**：通过 SEO、订阅（RSS/邮件）、分享与结构化内容（系列/专题）让内容自然扩散。

### 1.2 目标用户
- **访客读者**：通过搜索/分享进入，关心内容质量、检索效率、阅读体验。
- **回访读者**：希望快速看到更新、按主题追更、订阅。
- **站点管理员（作者/团队）**：需要高效编辑、发布、统计、运营。

### 1.3 内容支柱（建议）
- 技术笔记（工程实践/踩坑/架构）
- 产品与方法论（复盘/框架/工具链）
- 个人随笔（可选，若要更“人味儿”）

### 1.4 体验原则
- **读者优先**：加载快、排版好、导航清晰、搜索可用。
- **结构化胜过堆砌**：分类/标签/系列/归档并存，但要有主次。
- **默认可分享**：OpenGraph、可复制标题链接、代码块体验好。

## 2. 全站信息架构与页面树（Sitemap / Routes）

> 下述为“产品页面树”，实现时可映射为 Vue Router 路由。括号内为建议路径（可按实际后端路由调整）。

### 2.1 公共前台（Public）
- 首页（`/`）
- 文章
  - 文章详情（`/posts/:slug`）
  - 文章列表（`/posts`）
  - 分类页（`/category/:slug`）
  - 标签页（`/tag/:slug`）
  - 系列/专题页（`/series/:slug`）
  - 归档页（按时间）（`/archive`）
  - 搜索结果页（`/search?q=`）
- 页面（Pages）
  - 关于（`/about`）
  - 友链/推荐（`/links`，可选）
  - 使用/订阅（`/subscribe`：RSS、邮件订阅、社交账号）
- 订阅与机器可读
  - RSS（`/rss.xml`）
  - Sitemap（`/sitemap.xml`）
- 状态与兜底
  - 404（`/404` 或路由兜底）
  - 500（`/500`，可选）

### 2.2 管理后台（Admin）
> 后台建议独立布局与路由前缀，避免与公共前台样式耦合。

- 登录（`/admin/login`）
- 仪表盘（`/admin`）
- 内容管理
  - 文章列表（`/admin/posts`）
  - 新建/编辑文章（`/admin/posts/new`、`/admin/posts/:id/edit`）
  - 草稿箱（`/admin/drafts`，也可用筛选代替）
  - 分类管理（`/admin/categories`）
  - 标签管理（`/admin/tags`）
  - 系列/专题管理（`/admin/series`）
  - 页面管理（关于/友链等静态页）（`/admin/pages`）
- 媒体资源
  - 媒体库（图片/附件）（`/admin/media`）
- 互动与运营（可选）
  - 评论审核（`/admin/comments`）
  - 订阅/通知配置（`/admin/notifications`）
- 配置与系统
  - 站点设置（标题、描述、头像、社交链接）（`/admin/settings/site`）
  - SEO 设置（默认 meta、OG、robots）（`/admin/settings/seo`）
  - 用户与权限（若多人协作）（`/admin/users`）
  - 数据备份/导出（`/admin/backup`）

## 3. 顶层导航（IA 导航模型）

### 3.1 主导航（Header）建议
- 首页
- 文章（下拉：最新 / 分类 / 标签 / 归档 / 系列）
- 关于
- 搜索（图标入口，桌面端可内嵌输入）

> 注意：主导航不宜过长，**分类/标签/系列**建议在“文章”下拉或二级页承载。

### 3.2 次级导航/侧边信息（可选）
- 个人名片（头像、简介、社交链接）
- 热门文章 / 置顶文章
- 标签云 / 分类列表（可折叠）
- 订阅入口（RSS/邮件）

## 4. 首页（/）区块设计（Homepage Blocks）

> 首页的职责：**建立信任 + 快速发现内容 + 引导订阅/关注**。

推荐区块（从上到下）：
1. **Hero 区（首屏）**
   - 站点标题 + 一句话定位（你写什么、为何值得看）
   - CTA：订阅 / 查看最新文章 / 关于我
2. **置顶/精选（Featured）**
   - 3–6 篇：可手动置顶，或按阅读/收藏排序
3. **最新更新（Latest Posts）**
   - 列表式信息密度高：标题 + 摘要 + 日期 + 分类/标签
4. **内容入口（Topics）**
   - 2–6 个核心分类卡片：例如「工程」「产品」「工具」
5. **系列/专题入口（Series）**（若有）
   - 让读者按“课程/专题”方式阅读，降低跳出
6. **关于作者（Mini About）**
   - 简短介绍 + 链接到 `/about`
7. **订阅/联系方式（Subscribe / Contact）**
   - RSS、邮件订阅（如未来接入）、社交账号
8. **页脚（Footer）**
   - 版权、备案（如需）、站点地图、RSS、主题/暗色模式入口

## 5. 文章系统设计（Posts / Taxonomy / Reading Experience）

### 5.1 内容对象与关系（概念模型）
- **Post（文章）**
  - 状态：draft / published / scheduled / archived
  - 字段：title、slug、summary、content、cover、publishedAt、updatedAt、readingTime、wordCount
  - 关系：belongsTo Category；many-to-many Tags；optional Series
- **Category（分类）**：建议 1 层或最多 2 层，避免无限层级导致导航复杂
- **Tag（标签）**：用于横向关联，数量可多但需后台治理（合并/别名）
- **Series（系列/专题）**：提供连续阅读顺序（part 1/2/3）
- **Page（静态页）**：About、Links、Subscribe 等

### 5.2 URL 规范（建议）
- 文章详情：`/posts/{slug}`（稳定、利于 SEO）
- 分类：`/category/{slug}`；标签：`/tag/{slug}`；系列：`/series/{slug}`
- 归档：`/archive`（按年/月折叠）

> 若后端需要日期型 URL（如 `/2026/03/slug`）也可，但 slug-only 更利于重排与迁移。

### 5.3 文章列表页（/posts）结构
- 筛选/排序：
  - 分类、标签、系列（可选）
  - 排序：最新 / 最热（可后续）
- 列表项展示：
  - 标题、摘要、日期、阅读时长、分类/标签、封面（可选）
- 分页策略：
  - MVP：分页（page=1/2）
  - 后续：无限滚动（但对 SEO 不友好，需 SSR/预渲染配合）

### 5.4 文章详情页（/posts/:slug）阅读区块
- 标题区：标题 + 发布时间/更新时间 + 分类/标签 + 阅读时长
- **目录 TOC**（建议默认开启，长文必备）
- 正文内容（Markdown 渲染为主的方向更合理）
- 文章末尾：
  - 版权声明 / 转载说明
  - 上一篇/下一篇
  - 相关文章（同标签/同系列）
  - 评论区（可选：自建/第三方，如 Giscus）
  - 分享（复制链接、微博/推特等可选）

### 5.5 搜索（/search）
- MVP：站内标题/摘要搜索（前端过滤或后端简单检索）
- 进阶：全文检索（后端索引，或接入轻量搜索服务）

### 5.6 归档（/archive）
- 年 -> 月折叠
- 展示：标题 + 日期（无需摘要，信息密度优先）

## 6. 后台系统建议（Admin）

### 6.1 后台核心流程（作者视角）
1. 登录
2. 新建文章（标题/slug/摘要/封面）
3. 编辑正文（建议 Markdown 编辑器 + 预览）
4. 选择分类/标签/系列
5. 保存草稿 / 预览 / 发布（支持定时发布为加分项）
6. 发布后可再次编辑（记录更新时间，必要时保留修订历史）

### 6.2 后台页面清单与关键能力
- 仪表盘：
  - 最近发布、草稿数量、近 7 天访问（可后续接入统计）
  - 待处理事项（评论待审、标签待治理等）
- 文章管理：
  - 列表：状态筛选（草稿/已发布/定时）、搜索、批量操作（发布/下线/删除）
  - 编辑：自动保存草稿、预览（最好是与前台同渲染）
- 分类/标签/系列：
  - CRUD、slug 校验、合并/别名（标签尤其需要）
- 媒体库：
  - 上传、复制 URL、简单裁剪/压缩（进阶）
- 设置：
  - 站点信息、导航配置、首页模块开关（可选）、SEO 默认值

### 6.3 权限模型（若未来多人协作）
- Owner：全权限
- Editor：内容管理、媒体管理
- Viewer：只读后台数据

## 7. MVP 范围建议（分阶段交付）

### Phase 0：站点骨架（最快可见）
- 公共前台：`/`、`/posts`、`/posts/:slug`、`/about`、`404`
- 后台：`/admin/login`、`/admin/posts`、`/admin/posts/new|:id/edit`

### Phase 1：内容组织能力
- 分类/标签页（前台 + 后台管理）
- 归档页
- RSS + Sitemap

### Phase 2：体验与增长
- TOC、代码高亮、图片灯箱
- 搜索增强（至少标题/摘要可用）
- 相关文章、系列阅读

### Phase 3：运营与系统化
- 评论系统与审核
- 访问统计、热门文章
- 定时发布、修订历史、内容导入导出

## 8. 后续演进建议（Architecture / Product Evolution）

### 8.1 渲染策略（SEO 相关）
- 当前前端栈是 Vite SPA，**SEO 会受限**。
- 选项：
  1. **预渲染（Prerender/SSG）**：适合博客，成本较低（构建时生成静态 HTML）
  2. **SSR（服务端渲染）**：更强但复杂度更高

> 建议优先走 SSG/预渲染路线；文章内容天然适配静态化。

### 8.2 内容格式与编辑器
- 推荐以 **Markdown** 作为主内容格式：易迁移、生态成熟。
- 后台编辑器可选：Markdown + 预览（MVP）；后续再做富文本/块编辑。

### 8.3 结构化数据与分享
- OpenGraph（标题/摘要/封面图）
- 结构化数据（Article schema）
- 自动生成分享图（OG Image）作为后续增强

### 8.4 性能与可维护性
- 图片：压缩、响应式（srcset）、懒加载
- 代码分割：按路由拆包
- 缓存：静态资源长缓存 + 内容更新策略

### 8.5 国际化与多语言（可选）
- 多语言路由前缀：`/en/...`
- 内容层面：同一 slug 不同 locale 或独立 slug
